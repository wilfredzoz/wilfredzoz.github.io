输出精度的控制

  • 用m.n来控制数字的宽度和小数点后的精度，如在字符串的格式化中years=11,那么用“%5d” % years，就会表示为(默认右对齐)“ 11”。前面空了三个格子。

  • 又如money = 11.235，那么用“%7.2d” % money，就会表示为“ 11.24”，其中空两格是因为要求整体宽度为7（包括小数点）.2要求的是将小数保留最后两位。%.2d表示不限制宽度，只保留两位小数。

  • 在字符串的输入中，也可以用最大输出长度来限制字符的多少，如：%10.4s：表示总位数为10，取字符串的前4位输出

又如:🎃^10.4s，控制总输出宽度为10，输出字符串的前四位

ps：如果设置的宽度比数值的真实宽度要窄，那么会直接截掉小数点最后的数字，不会对其做四舍五入。如果对于整数来说，设置的宽度比数值的真实宽度要窄，和没有设置宽度是一样的（五舍六入？？？？疑问点？？？？？？？，“%5.2f” % 11.346与“%5.2f” % 11.646的结果不同）

pyspark的数据输入

pyspark在通过SparkContext类完成数据的输入之后，会得到一个RDD类对象，RDD全程弹性分布式数据集（Resilient Distributed Datasets）,

pyspark针对数据集的处理，都是以RDD对象作为载体，即

  • 数据储存在RDD内
  • 各类数据的计算方法，也都是RDD的成员方法
  • RDD的数据计算方法，返回值依旧是RDD对象

pyspark支持通过SparkContext类对象的parallelize成员方法，将

  • list
  • tuple
  • str
  • dict
  • set

reduceByKey()方法（算子）

reduceByKey()方法是spark中针对Key-value型(必须是元组类型的KV数据)的数据进行分组运算你的一种方法，这个方法首先根据传输的数据中的key进行分类，然后再根据传入的函数，对value进行聚合操作，传入的函数只负责聚合操作，和分类没有关系。

注意reduceByKey()方法中传入函数的要求：[v, v] -> v ，要求两个传入参数，返回值类型和传入参数类型相同

由函数进行聚合操作有以下的规则：若分类之后，某一类key对应的value的数值和顺序分别为1，2，3，4，5，这五个数字按照以下方式进行运算

graph LR
    A[1] --> |a|G["func(a,b)"]
    B[2] --> |b|G
    C[3] --> |b|H["func(a,b)"]
    D[4] --> |b|I["func(a,b)"]
    E[5] --> |b|J["func(a,b)"]

    G --> |a|H
    H --> |a|I
    I --> |a|J
    J --> K[最终输出]

上述不同的结果是由于聚合函数的操作顺序不一致导致的，rdd_1和rdd_2都将kv类型的数据分为“a”和“c”两类，但是再rdd_1和rdd_2中同样的key所对应的value顺序是不同的，rdd_1进行的操作是（'a', (1-2)-5=-6），rdd_2进行的操作是（'a', (2-1)-5=-4），最终他们的key=‘a’所计算出来的值是不一样的。

filter()算子

rdd.filter()算子可以用来对rdd中的数据进行过滤，func为[T] -> bool ,传入一个参数，返回值为bool，rdd.filter()算子中的函数挨个对数据进行计算，如果返回值为True则保留该数据，False则过滤掉该数据

distinc（）算子

rdd.distinc（）算子用来对rdd中的数据进行去重（传入rdd的列表中的元素），括号中无需传入参数（除了numPartitions，涉及分布式的知识，先不考虑）

sortBy（）算子

sortBy（）算子可以用来对rdd中的数据进行排序，他有三个参数，分别是func：[T] ->U，ascnding，和numPartitions。func：[T] ->U指的是sortBy（）算子的依据函数的返回值进行排序，函数传入一个参数，计算得到不同类型的返回值，依据返回值进行排序，ascnding表示的是是否升序排列，可选Ture和False，numPartitions为分布式的知识，一般令其为1
