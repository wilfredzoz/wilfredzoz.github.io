hexo博客中mathjax公式格式

发表于 2024-06-22 更新于 2025-03-20 分类于 Hexo
博客中公式显示的问题

在输入公式中,要注意一些规范

在hexo博客中:

  • 若公式中需要输入两个连起来的{{}},在相同的符号之间加上一个空格,变成{ {} } .否则执行hexo g 的时候会出错
  • 在输入矩阵的LATEX公式时,表达式中不要留出多余的空行,否则会出错
  • 上述两个问题都会报错:Nunjucks Error: _posts/文件名.md [Line xx, Column xx] expected variable end

在Github中:

  • 在 \(\LaTeX{}\) 中，如果要输入行内代码用一个 $ 括起来:$+LATEX代码+$ , 注意 $不能与字符或符号相连,否则GITHUB无法显示

  • 注意软换行和硬换行，在Typora中，ENTER键为硬换行，在源码内查看其实是换了两行，但是在预览区域可能无法清晰的看出来；而SHIFT+ENTER为软换行，在源码中换一行

  • 在输入代码块时，一定要注意以下要求：

空行

【顶格】 $$

latex代码

【顶格】 $$

空行

上下只能空一行，否则GITHUB不能读出来

  • 在写行内代码时： $与代码之间不能有空格

    eg：[空格]$LATEX代码$[空格]

  • 在写下标的时候记得在_后加一个空格，否则Github可能不会显示出公式来，原因未知

    eg: $X_[空格]n$

  • 若有比较符号：0 < k < n，在比较的左右加上空格

    eg： $0[空格]<[空格]k[空格]<[空格]n$

  • 求和符号的写法需要改变：

\sum_ {t=1}^{n-k}： \(\sum\limits_ {t=1}^{n-k}\)

\sum_ {t=1}^{n-k}： \(\sum\nolimits_ {t=1}^{n-k}\)

若不加\limits 或 \nolimits Github可能会显示出错

\(\begin{matrix} A.\frac{\sum\limits_{i=1}^{n}(x_{i}-\bar{x})(y_{i}-\bar{y})}{\sum\limits_{i=1}^{n}(y_{i}-\bar{y})^2} && B.\frac{\sum\limits_{i=1}^{n}(x_{i}-\bar{x})(y_{i}-\bar{y})}{\sum\limits_{i=1}^{n}x_{i}^2} \\ C.\frac{\sum\limits_{i=1}^{n}(x_{i}-\bar{x})(y_{i}-\bar{y})}{\sum\limits_{i=1}^{n}(x_{i}-\bar{x})^2} && D.\frac{\sum\limits_{i=1}^{n}(x_{i}-\bar{x})^2}{\sum\limits_{i=1}^{n}(y_{i}-\bar{y})^2} \end{matrix}\)
