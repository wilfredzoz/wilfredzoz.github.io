关系型数据库

关系型数据库是一种以关系模型为理论基础的数据库系统，它通过二维表的形式存储数据，并且这些表之间通过关系进行连接。

表一：

id name job dept_id
1 金庸 总裁 2
2 张无忌 项目经理 1
3 杨逍 开发 3

表二：

id name
1 研发部
2 总经办
3 财务部

表一和表二为两张二维表格，且两张表格通过dept_id连接起来，两张表可以相互查找，这些表就称为关系型数据库表。

关系型数据库表有以下特点：

  • 使用表存储数据，格式统一，便于维护。

  • 使用SQL语言操作，标准统一，使用方便。

通过表结构存贮数据的数据库称为关系型数据库，不通过表结构存贮数据的称为非关系型数据库

例如最长50个字符的用户名，因为每个人的用户名长度不同，若使用char，那么剩余的空间将是多余的，所以使用varchar(50)更好

再例如性别，使用性能较高的char(1)即可。

where与having的区别

执行时机不同:where是分组之前进行过滤，不满足where条件，不参与分组;而having是分组之后对结果进行过滤。

判断条件不同:where不能对聚合函数进行判断，而having可以。

执行顺序:where>聚合函数>having。

分组之后，查询的字段一般为聚合函数和分组字段，查询其他字段无任何意义。也就是说只有在select之后的字段或者聚合函数，才能进行group by，因为只有从表中选出字段，才能对这些字段进行分组。

排序查询

排序方式 ASC: 升序(默认值) DESC: 降序 注意: 如果是多字段排序，当第一个字段值相同时，才会根据第二个字段进行排序。

排序查询案例

在emp表中通过排序查询，查询以下信息：

  • 根据年龄对公司的员工进行升序排序

order by age默认为升序排列

  • 根据入职时间，对员工进行降序排序
  • 根据年龄对公司的员工进行升序排序，年龄相同，再按照入职时间进行降序排序

图中看出年龄升序排列，若年龄相等，则按照入职时间降序排列。

分页查询

注意

  • 起始索引从 0 开始，起始索引 = (查询页码 - 1) * 每页显示记录数。

  • 分页查询是数据库的方言，不同的数据库有不同的实现，MySQL中是LIMIT。

  • 如果查询的是第一页数据，起始索引可以省略，直接简写为limit 10。

分页查询案例

在emp表中通过分页查询，查询以下信息：

  • 查询第1页员工数据，每页展示10条记录
  • 查询第2页员工数据，每页展示10条记录

计算起始索引：起始索引= (查询页码 - 1) * 每页显示记录数。计算起始索引=10。

DQL语句案例

在emp表中通过DQL语句，查询以下信息：

  • 查询年龄为20,21,22,23岁的员工信息。
  • 查询性别为男，并且年龄在20-40岁(含)以内的姓名为三个字的员工。
  • 统计员工表中，年龄小于60岁的，男性员工和女性员工的人数。
  • 查询所有年龄小于等于35岁员工的姓名和年龄，并对查询结果按年龄升序排序，如果年龄相同按入职时间降序排序。
  • 查询性别为男，且年龄在20-40岁(含)以内的前5个员工信息，对查询的结果按年龄升序排序，年龄相同按入职时间升序排序。

DQL的执行顺序

DQL的规范编写顺序为：

flowchart LR

    字段列表"] -->B["FROM
    表名列表"] -->C["WHERE
    条件列表"] --> D["GROUP BY
    分组字段列表"] --> E["HAVING
    分组后的条件列表"] --> F["ORDER BY
    排序字段列表"] --> G["LIMIT
    分页参数"]
